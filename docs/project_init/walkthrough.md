# Phase 1 & 2: プロジェクト初期化と認証機能 実装結果

これまでに実装および完了した内容のウォークスルーです。

## 1. プロジェクト基盤の構築 (Phase 1)
- **Next.js 16 (App Router)** の環境を構築しました。
- **Tailwind CSS v4** を導入し、UIデザインガイドラインに沿ったブランドカラー（背景、文字、アクセント色）をセットアップしました。
- `@serwist/next` を使用し、インストール可能な **PWA (Progressive Web App)** 機能を設定しました。

## 2. 認証・ユーザー管理基盤の実装 (Phase 2)
`.env.local` に設定いただいた Supabase キーを用いて、安全な認証機能を構築しました。

### 年齢確認とログインフロー
- `/login` ページにアクセスすると、まず年齢確認が表示されます。
- **12歳以下の場合**:
  - GitHubログインはできず、「ひみつのじゅもん」を入力する画面になります。
  - バックエンドでは Supabase のメール＆パスワード認証を応用し、「`kid_<じゅもん>@truckers.jp`」のようなダミーアドレスとパスワードとして「ひみつのじゅもん」を利用する仕組みになっています。これにより、通常のユーザーと同じセキュアなインフラにデータを保存・復元できます。
- **13歳以上の場合**:
  - GitHub でのログインボタンが表示されます（Supabase OAuth連携）。

### 匿名ユーザー名の生成機能
- `src/utils/nameGenerator.ts` を作成し、「形容詞＋動物名＋数字（例: かわいいネコ4821）」で構成される完全匿名のユーザー名をランダムに生成するロジックを実装しました。（※この生成関数はPhase3でユーザー情報を保存する際に呼び出して利用します）

---

> [!TIP]
> 動作確認を行う場合は、ローカル環境で `npm run dev` を実行し、ブラウザで `http://localhost:3000/login` にアクセスしてログインUIや年齢確認の切り替えをお試しいただけます。

> [!NOTE]
> Supabase側で **Email Auth (メールとパスワードによる認証)** が有効になっていることをご確認ください（13歳未満のシークレットコードでログインする際に使用します）。

すべて問題なくビルドが通り、GitHubリポジトリ（`main` ブランチ）にプッシュ済みです。
